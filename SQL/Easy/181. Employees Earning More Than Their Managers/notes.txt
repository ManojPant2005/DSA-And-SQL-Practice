# LeetCode 181 - Employees Earning More Than Their Managers

## Problem Summary

We have an Employee table containing:

* Employee ID
* Employee Name
* Salary
* Manager ID

We need to find employees whose salary is greater than their manager's salary.

---

## Key Observation

The manager information is stored in the same Employee table.

There is no separate Manager table.

Example:

| id | name | salary | managerId |
| -- | ---- | ------ | --------- |
| 1  | Joe  | 70000  | 3         |
| 3  | Sam  | 60000  | NULL      |

Joe's manager is Sam because:

```sql
Joe.managerId = Sam.id
```

Therefore:

```text
70000 > 60000
```

Joe should be returned.

---

## Concept Learned: Self Join

### What is a Self Join?

A Self Join is a join where a table is joined with itself.

It is useful when records in a table are related to other records in the same table.

Common Examples:

* Employee → Manager
* Category → Parent Category
* Folder → Parent Folder
* Comment → Parent Comment

---

## Query

```sql
SELECT e.name AS Employee
FROM Employee e
JOIN Employee m
    ON e.managerId = m.id
WHERE e.salary > m.salary;
```

---

## Query Breakdown

### Step 1: Create Aliases

```sql
Employee e
Employee m
```

* e = Employee
* m = Manager

---

### Step 2: Match Employee with Manager

```sql
ON e.managerId = m.id
```

Example:

```text
Joe.managerId = 3
Sam.id = 3
```

Result:

```text
Joe → Sam
```

---

### Step 3: Compare Salaries

```sql
WHERE e.salary > m.salary
```

Example:

```text
70000 > 60000
```

True

Therefore Joe is selected.

---

## Output

| Employee |
| -------- |
| Joe      |

---

## Interview Questions

### Q1. Why is a Self Join used?

Because manager information exists in the same Employee table.

---

### Q2. What is the join condition?

```sql
e.managerId = m.id
```

---

### Q3. What type of join is used?

```sql
INNER JOIN
```

Because only employees having managers need to be compared.

---

## Key Takeaway

Whenever a table references itself, think about using a Self Join.

Pattern:

```sql
SELECT ...
FROM TableName t1
JOIN TableName t2
    ON t1.SomeColumn = t2.Id
```
